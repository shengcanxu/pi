# Agent Instructions

## Changelog

Keep changelog entries focused on user-visible changes.

- Prefer a top `## [Unreleased]` section while work is in progress.
- Only mention features/fixes users will notice.
- Do **not** list documentation-only updates, test-only updates, or internal refactors unless they directly change user behavior.
- Group release notes under concise headings (e.g. `Added`, `Changed`, `Fixed`).
